Section 1~22%

Designing Data Processing Systems

Architect for security and compliance, reliability and fidelity, flexibility and portability, and clean migrations.

Security, governance & compliance

Apply IAM and org policies, encrypt with CMEK where required, and handle PII with Cloud DLP. Design project/dataset/table architecture for governance and data residency. Separate dev and prod environments.

Reliability, fidelity & migrations

Prepare and clean data with Dataform, Dataflow, and Cloud Data Fusion; orchestrate and monitor pipelines; design for fault tolerance and DR; validate data and reason about ACID needs. Plan migrations with BigQuery Data Transfer Service, Database Migration Service, Datastream (CDC), and Transfer Appliance for offline bulk.

⚠️ Exam Trap: Continuous, low-downtime database replication to Google Cloud = Datastream (CDC) or DMS — not Storage Transfer Service (objects) or Data Fusion (batch ETL).

The migration & transfer tool picker

The exam constantly asks "which mover?" — map the data shape to the tool:

  • Datastream — continuous CDC replication from operational databases with minimal downtime.
  • Database Migration Service — homogeneous/heterogeneous database migrations (e.g. to Cloud SQL/AlloyDB).
  • BigQuery Data Transfer Service — scheduled loads from SaaS and other warehouses into BigQuery.
  • Storage Transfer Service — bulk object transfer (other clouds, on-prem) into Cloud Storage.
  • Transfer Appliance — offline shipment of petabyte-scale data when the network can't.
MOVE Database → DMS/Datastream; Objects → Storage Transfer; Too big for the wire → Transfer Appliance.

🧠 Warm-Up Quiz

Q1. Discover and mask PII in a dataset before consumer analytics, scalably. Which combo?

Reveal answer

Dataflow + Cloud DLP to mask/redact, then write to BigQuery.

Scalable de-identification while preserving useful data.

Q2. Continuously replicate an operational database into BigQuery with minimal downtime. Which tool?

Reveal answer

Datastream (CDC).

Streams change data continuously; STS is for objects, Data Fusion for batch.

Q3. 800 TB of archives must move to Cloud Storage but the WAN link would take months. What do you use?

Reveal answer

Transfer Appliance.

Offline, shippable device for petabyte-scale data when the network is the bottleneck.

Q4. Which key model lets you control rotation and revocation of the keys protecting a dataset?

Reveal answer

CMEK via Cloud KMS.

Customer-managed keys give you rotation/revocation control; Google-managed keys don't.

📝 Exam-Style Practice

Q1. An operational MySQL database must be continuously replicated into BigQuery for near-real-time analytics, with minimal downtime and no application changes. Which service fits?

  • A) Storage Transfer Service syncing exported dump files.
  • B) Cloud Data Fusion running a nightly batch ETL.
  • C) Datastream change-data-capture into BigQuery.
  • D) A scheduled BigQuery query that pulls from the database.
Reveal answer & explanation

Correct: C. Datastream streams change data continuously with minimal source impact and downtime — the purpose-built CDC service into BigQuery.

A: Storage Transfer moves objects, not live DB changes. B: Nightly batch isn't near-real-time. D: BigQuery can't directly poll an operational MySQL continuously like CDC.

Q2. Before analysts can access a customer dataset, the team must detect and redact PII at scale while keeping the rest of the records usable. What pipeline best achieves this?

  • A) A Dataflow pipeline calling Cloud DLP to de-identify, writing results to BigQuery.
  • B) A BigQuery view that hides columns named like PII.
  • C) IAM that restricts who can read the raw dataset.
  • D) CMEK encryption applied to the dataset.
Reveal answer & explanation

Correct: A. Dataflow + Cloud DLP detects many PII types (even in free text) and de-identifies at scale, producing analytics-safe data.

B: Name-based column hiding misses PII in text and free fields. C/D: Access controls and encryption limit who/what but don't remove PII from the data analysts see.

Q3. A company must move 600 TB of archived files from an on-prem NAS to Cloud Storage, but their internet link would take many months to transfer it. What is the right approach?

  • A) Storage Transfer Service over the existing internet connection.
  • B) gsutil/gcloud storage parallel uploads from the NAS.
  • C) Transfer Appliance shipped to Google for ingestion.
  • D) Datastream CDC from the file system.
Reveal answer & explanation

Correct: C. When the network can't move the volume in a reasonable time, Transfer Appliance ships the data physically for offline ingestion.

A/B: Both still depend on the slow link. D: Datastream is database CDC, not bulk file transfer.
