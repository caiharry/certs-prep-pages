Section 4~15%

Preparing & Using Data for Analysis

Make data fast for BI, ready for ML, and safely shareable.

BI acceleration & access

Speed BI with BI Engine and materialized views; pre-calculate fields; troubleshoot slow queries. Connect tools like Connected Sheets (spreadsheet access to BigQuery) and Looker Studio. Protect data with column-level security, data masking, IAM, and DLP.

⚠️ Exam Trap: Business users who want to analyse BigQuery data in a spreadsheet need Connected Sheets — Looker Studio is a dashboard, not a sheet.

Data for AI/ML and sharing

Prepare features for training/serving (incl. BigQuery ML), and prepare unstructured data for embeddings and RAG. Share governed data with Analytics Hub (publish/subscribe to datasets) rather than copying.

⚠️ Exam Trap: Share BigQuery data across teams with least operational overhead via Analytics Hub, not by replicating tables into each project.

Fine-grained access & masking

Share data without over-exposing it. Use authorized views / authorized datasets to expose only chosen rows/columns, column-level security (policy tags via Dataplex) to gate sensitive fields, dynamic data masking to show redacted values to some roles, and row-level security to scope each consumer to their slice. Layer Cloud DLP for detection.

⚠️ Exam Trap: "Analysts may query the table but must never see the raw card number" = column-level security / dynamic masking, not a separate copied table.

🧠 Warm-Up Quiz

Q1. Non-technical users want to wrangle data with a GUI then analyse it in a spreadsheet. Which combo?

Reveal answer

Dataprep + Connected Sheets.

GUI wrangling; spreadsheet access to BigQuery results.

Q2. Let many teams query each other's BigQuery data with minimal ops and no duplication. Which feature?

Reveal answer

Analytics Hub (BigQuery sharing).

Publish/subscribe to shared datasets without copies.

Q3. Analysts may query a table but must never see the raw SSN column. What enforces this?

Reveal answer

Column-level security (policy tags) or dynamic data masking.

Restricts or masks the sensitive column while leaving the rest queryable — no copied table needed.

Q4. A query is slow and over-budget for a repeated aggregation. Cheap BI accelerator?

Reveal answer

BI Engine and/or a materialized view.

Both cache/precompute results so the heavy aggregation isn't rerun from scratch each time.

📝 Exam-Style Practice

Q1. Several teams need to query a central BigQuery dataset, but the data owner wants to avoid copying tables into each project and minimise operational overhead. What should they use?

  • A) A scheduled query that replicates the tables into each team's project nightly.
  • B) Analytics Hub to publish the dataset and let teams subscribe in place.
  • C) Export to Cloud Storage and have each team import it.
  • D) Grant every analyst Owner on the source project.
Reveal answer & explanation

Correct: B. Analytics Hub shares datasets via publish/subscribe with no copying and minimal ops — consumers query the live data in place.

A/C: Replication and exports duplicate data and add maintenance. D: Owner is wildly over-privileged and still doesn't solve sharing cleanly.

Q2. Analysts must run queries over a customer table, but policy says they must never see the raw credit-card column, while still using the other fields. What is the cleanest control?

  • A) Maintain a second copy of the table with the column removed for analysts.
  • B) Apply column-level security (policy tags) and/or dynamic data masking to the sensitive column.
  • C) Revoke all analyst access to the dataset.
  • D) Encrypt the whole table with CMEK.
Reveal answer & explanation

Correct: B. Column-level security and dynamic masking restrict or redact just the sensitive field while keeping the rest queryable — no duplicate tables to maintain.

A: A second copy is extra maintenance and drift risk. C: Revoking access blocks the legitimate analysis. D: CMEK protects keys, not which columns a role can read.

Q3. Business users want to explore and pivot BigQuery data inside a familiar spreadsheet, against billions of rows, without exporting samples. Which tool fits?

  • A) Looker Studio dashboards.
  • B) Connected Sheets, which queries BigQuery directly from a spreadsheet.
  • C) Downloading a CSV extract into the spreadsheet.
  • D) A materialized view exposed via the API.
Reveal answer & explanation

Correct: B. Connected Sheets lets users analyse full BigQuery datasets from a spreadsheet without extracting samples.

A: Looker Studio is a dashboard, not a spreadsheet. C: CSV extracts cap at sample sizes and go stale. D: A materialized view speeds queries but isn't a spreadsheet interface.
