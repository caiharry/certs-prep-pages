UK CGP Revision Style

GCP Cloud Architect

High-yield revision deck ordered by the official exam guide. Pillars, decision matrices, and the traps the examiners love.

🏛️ The big picture: The PCA exam rewards judgement over recall. Anchor every answer in the Well-Architected Framework pillars — operational excellence, security, reliability, performance, cost, sustainability — and the explicit business constraint in the scenario. Read the four case studies (Altostrat Media, Cymbal Retail, EHR Healthcare, KnightMotives Automotive) before exam day.
Section 1 ~25%

Designing & Planning the Architecture

Translate business + technical requirements into a design. This is a quarter of the exam.

Requirements & trade-offs

  • Functional = what it does; non-functional = how well (latency, RTO/RPO, availability).
  • Always tie design to the stated KPI/ROI and BCP.
  • Workload disposition: rehost, replatform, refactor, rebuild, retire (build/buy/modify/deprecate).

Compute selection

  • Cloud Run: stateless containers, scale-to-zero, fastest to ship.
  • GKE: complex microservices, fine control, sidecars/mesh.
  • Compute Engine: lift-and-shift, licensing, GPUs, VMware Engine.
  • Spot VMs: fault-tolerant batch only.

Modern AI building blocks

  • Model Garden: catalog of foundation + open models.
  • Vertex AI Agent Builder: RAG/search/agents.
  • AI Hypercomputer: large-scale GPU/TPU training.
  • Gemini Cloud Assist: design/ops help inside the console.

⚠️ Exam Trap: Don't pick GKE when the scenario stresses minimal ops / fastest delivery — that's Cloud Run.
Section 2 ~17.5%

Managing & Provisioning Infrastructure

Network topologies, storage, compute, and Agent Platform ML workflows.

Hybrid connectivity tiers

  • HA VPN: 99.99% SLA, IPsec over internet, quick.
  • Dedicated Interconnect: 10/100G, private, lowest latency, needs colo.
  • Partner Interconnect: via SP when no colo presence.
  • Cloud Router = dynamic BGP for all of the above.

Shared VPC vs Peering

  • Shared VPC: one host project, central network admin, many service projects.
  • VPC Peering: connect two VPCs, non-transitive, decentralised.
  • PSC: private, one-way access to a published service.

Backup & DR tiers

  • Backup/restore: cheapest, highest RTO.
  • Cold/warm standby: lower RTO, more cost.
  • Hot/multi-region active-active: near-zero RTO/RPO, priciest.
⚠️ Exam Trap: Interconnect is not encrypted by default — add HA VPN over it or MACsec if encryption-in-transit is required.
⚠️ Exam Trap: VPC Peering is non-transitive: A–B and B–C does not give A–C.
Section 3 ~17.5%

Designing for Security & Compliance

Least-privilege IAM, the resource hierarchy, encryption, and regulatory mapping.

Resource hierarchy & policy

  • Org → Folders → Projects → Resources; IAM & org policies inherit downward.
  • Set guardrails high (org policy) e.g. restrict regions, disable SA key creation.
  • Separation of duties: split key admin from key user.

Encryption key options

  • Google-managed: default, zero effort.
  • CMEK (Cloud KMS): you control rotation/revocation.
  • EKM: keys stay in external/on-prem HSM.
  • Confidential Computing: encrypt data in use.

Access boundaries

  • VPC Service Controls: perimeter against data exfiltration (serverless APIs).
  • IAP / Chrome Enterprise Premium: context-aware, identity-based access — no VPN.
  • WIF: federate external identities without SA keys.
⚠️ Exam Trap: To stop data exfiltration from a serverless product (BigQuery, Pub/Sub, GCS) use VPC Service Controls, not firewall rules.
⚠️ Exam Trap: "Keys must never leave our HSM" = Cloud EKM; "we generate keys then import" = Cloud KMS import.
Section 4 ~15%

Analyzing & Optimizing Processes

Technical and business processes — SDLC, DR, change, and cost.

DR metrics

  • RTO = how long to recover; RPO = how much data you can lose.
  • Tighter RTO/RPO → more replication → more cost.
  • Test DR with game-days; document runbooks.

Cost optimisation levers

  • CUDs (committed use) & sustained-use discounts.
  • Spot VMs for fault-tolerant batch.
  • Right-size with Active Assist / Recommender.
  • Lifecycle policies on storage; autoscaling.
CapEx/OpEx Cloud shifts spend from CapEx (buy hardware up front) to OpEx (pay-as-you-go) — a frequent business-process answer.
⚠️ Exam Trap: Committed-use discounts are resource commitments (vCPU/RAM), not all-or-nothing per VM — they survive resizing.
Section 5 ~12.5%

Managing Implementation

Advise teams and interact with Google Cloud programmatically.

Tooling

  • Apigee: full API management (quotas, monetisation, versioning).
  • API Gateway: lightweight gateway for serverless.
  • Terraform / Infrastructure Manager: declarative IaC.

Dev access

  • Cloud Shell / Code: browser dev + emulators.
  • SDKs: gcloud, gsutil (now gcloud storage), bq.
  • Cloud Emulators: Bigtable, Spanner, Pub/Sub, Firestore for local test.
⚠️ Exam Trap: Apigee vs Cloud Endpoints/API Gateway: choose Apigee when you see monetisation, developer portal, or complex policy needs.
Section 6 ~12.5%

Solution & Operations Excellence

Observe, release safely, and prove resilience.

Observability stack

  • Cloud Monitoring: metrics, dashboards, alerting policies.
  • Cloud Logging: logs, log-based metrics, sinks.
  • Cloud Trace / Profiler: latency & CPU/heap profiling.

Release & resilience

  • Progressive delivery: canary, blue/green, rolling.
  • SLI/SLO/error budget gate release velocity.
  • Prove reliability with chaos engineering, load & pen testing.
⚠️ Exam Trap: Operational-excellence questions favour automation + measurable SLOs over heroic manual ops.
