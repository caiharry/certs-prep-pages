Section 5~12.5%

Managing Implementation

Advise dev and ops teams on deployment, and interact with Google Cloud programmatically.

API management & deployment

Apigee is full API management — quotas, key management, developer portal, monetization, analytics across hybrid backends. API Gateway is a lightweight gateway for serverless. Use load/unit/integration testing frameworks and Gemini Cloud Assist for guidance during rollout.

⚠️ Exam Trap: Choose Apigee when you see a developer portal, monetization, or complex policy needs; API Gateway is the lighter serverless option.

Programmatic access & IaC

Cloud Shell, Cloud Code, and Cloud Shell Editor for dev; SDKs gcloud, gcloud storage (formerly gsutil), and bq; Cloud Emulators (Bigtable, Spanner, Pub/Sub, Firestore) for local testing. Provision declaratively with Terraform or Infrastructure Manager — never click-ops for reproducible estates.

Advising teams & interacting with support

An architect also shapes how teams work: recommend testing strategies (unit/integration/load), code-review and trunk-based development, and a clear environment promotion path. Know the operational levers too — Cloud Quotas (request increases before launch spikes), the right support tier for the SLA, and using Gemini Cloud Assist to accelerate troubleshooting and design reviews.

⚠️ Exam Trap: "A planned launch will exceed current API limits" = request a quota increase ahead of time, not retry-on-failure during the event.

🧠 Warm-Up Quiz

Q1. You must expose partner APIs with quotas, key issuance, a developer portal, and analytics. Which product?

Reveal answer

Apigee.

Full lifecycle API management; API Gateway lacks the portal/analytics/monetization.

Q2. Reproducible, version-controlled infrastructure across environments. What approach?

Reveal answer

Infrastructure as Code (Terraform / Infrastructure Manager).

Declarative, repeatable, auditable — preferred over manual console changes.

Q3. A simple internal REST API on Cloud Functions just needs auth and basic routing — not a portal or monetization. Apigee or API Gateway?

Reveal answer

API Gateway.

It's the lightweight gateway for serverless backends; Apigee is overkill without portal/analytics/monetization needs.

Q4. A big product launch will far exceed your current API request limits. What do you do beforehand?

Reveal answer

Request a quota increase in advance via Cloud Quotas.

Quotas should be raised ahead of a known spike, not discovered when requests start failing.

📝 Exam-Style Practice

Q1. A company exposes APIs to external partners and needs quotas, API-key issuance, a self-service developer portal, monetization, and cross-backend analytics. Which product should the architecture use?

  • A) API Gateway in front of the serverless backends.
  • B) Apigee API management.
  • C) A Global External Application Load Balancer with Cloud Armor.
  • D) Cloud Endpoints with an OpenAPI spec.
Reveal answer & explanation

Correct: B. Apigee is full-lifecycle API management: developer portal, monetization, quotas, key management, and analytics across hybrid backends.

A/D: API Gateway and Endpoints are lightweight and lack the portal/monetization/analytics suite. C: A load balancer routes and protects traffic but isn't API management.

Q2. A marketing event is expected to drive 20× normal traffic to a service that calls several Google Cloud APIs. The architect wants to avoid throttling during the event. What is the right action?

  • A) Add aggressive client-side retries to ride through any limit errors.
  • B) Request the needed quota increases ahead of the event via Cloud Quotas.
  • C) Spread calls across several projects to multiply the quota.
  • D) Cache all API responses indefinitely to avoid calling the APIs.
Reveal answer & explanation

Correct: B. Known spikes call for raising quotas in advance so the service isn't throttled during the event.

A: Retries against an exceeded quota just amplify load. C: Splitting across projects to dodge limits is an abuse pattern. D: Indefinite caching serves stale data and doesn't fit dynamic calls.

Q3. Teams keep creating environments inconsistently by clicking in the console, causing drift between dev and prod. What should the architect mandate for reproducibility?

  • A) A wiki page documenting the console steps to follow.
  • B) Declarative Infrastructure as Code (Terraform / Infrastructure Manager) in version control.
  • C) Screenshots of the correct console settings shared with teams.
  • D) Granting only senior staff console access.
Reveal answer & explanation

Correct: B. IaC makes environments declarative, versioned, and reproducible, eliminating console drift.

A/C: Docs and screenshots still rely on manual, drift-prone steps. D: Limiting who clicks doesn't make creation repeatable.
