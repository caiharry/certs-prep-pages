Section 5~21%

Incident Response

Contain and investigate incidents, automate response with SOAR playbooks, and run the case lifecycle.

Contain & investigate

Scope incidents with Logs Explorer, Log Analytics, BigQuery, and SecOps. Isolate affected services (restrict the network) to stop spread while preserving the instance for forensics — don't delete evidence. Analyse artifacts (hash/IP/URL/binary) enriched by GTI, and perform root-cause analysis in the SecOps SIEM.

⚠️ Exam Trap: Contain by network-isolating a compromised instance, not by deleting it — deletion destroys forensic evidence.

SOAR playbooks & case management

Automate repetitive, well-understood response steps in SOAR playbooks (enrichment, containment, notifications), keeping humans for judgement calls. Run the case management lifecycle: assign cases to response stages, escalate efficiently, and ensure clean handoffs.

Playbook design: triggers, integrations & human-in-the-loop

A good SOAR playbook is more than a script — it's an orchestrated flow. Know the moving parts:

  • Trigger — what starts it: a new alert/case, a tag, or a specific detection. Scope triggers tightly so the right playbook runs on the right case.
  • Integrations & actions — connectors to EDR, IdP, firewalls, ticketing; actions enrich (GTI lookup), contain (isolate host, disable user), and notify.
  • Human-in-the-loop step — for irreversible or high-impact actions (wiping a host, locking an exec), insert a manual approval gate instead of full automation.
  • Automate the boring, escalate the judgement — fully automate repetitive deterministic steps; keep analysts for the calls that need context.
⚠️ Exam Trap: Don't fully automate a destructive or high-blast-radius action. The expected answer adds a human approval gate in the playbook, not unattended execution.

Case lifecycle, SLAs & post-incident

The case is the unit of work in SecOps SOAR. Drive it through a clean lifecycle and the metrics that prove it:

  • Triage → investigate → contain → eradicate → recover — assign cases to stages and owners so nothing stalls.
  • MTTD / MTTR & SLAs — track time-to-detect and time-to-respond; SLA breaches drive escalation and staffing decisions.
  • Evidence & chain of custody — keep artifacts, timelines, and analyst actions attached to the case for audit and any legal hold.
  • Post-incident review — feed lessons back into new detections, reference lists, and playbook tweaks so the same incident is cheaper next time.
PICERL Prepare, Identify, Contain, Eradicate, Recover, Lessons learned — the IR loop the case lifecycle mirrors.

🧠 Warm-Up Quiz

Q1. A VM is actively talking to a C2 server. Best immediate containment that preserves evidence?

Reveal answer

Network-isolate the instance.

Stops C2/lateral movement while keeping the instance for forensics.

Q2. Repetitive phishing-alert triage (enrich, disable session, notify) should be handled how?

Reveal answer

A SOAR playbook.

Automates consistent multi-step response.

Q3. A playbook step would wipe a production host. How should it be designed?

Reveal answer

Insert a human approval gate before the destructive action.

High-blast-radius/irreversible steps need human-in-the-loop, not unattended automation.

Q4. Leadership wants to know how long it takes from alert to containment. Which metric do you report?

Reveal answer

MTTR (mean time to respond/remediate).

MTTR captures alert-to-response duration; MTTD captures time to detect.

📝 Exam-Style Practice

Q1. A compromised Compute Engine VM is beaconing to a known C2 endpoint and may be moving laterally. Forensics will need disk and memory afterward, and the workload owner has not yet been reached. What is the correct first containment step?

  • A) Delete the instance immediately so the attacker loses their foothold and the beaconing stops
  • B) Apply a deny-all firewall policy to isolate the instance's network while leaving it running
  • C) Shut the instance down so it stops communicating until the owner approves further action
  • D) Rotate the service-account keys the VM uses and leave its network connectivity in place
Reveal answer & explanation

Correct: B. Network isolation via a deny-all policy halts C2 and lateral movement instantly while keeping the running instance — and its volatile memory — intact for forensics.

A: Deleting the VM destroys the very disk and memory evidence forensics needs. C: Shutting down loses volatile memory artifacts and can trigger anti-forensic or destructive malware on halt. D: Rotating keys may limit credential abuse but leaves the C2 channel and lateral movement fully open.

Q2. Your SOC handles dozens of identical low-risk malware-quarantine alerts daily: enrich the hash with GTI, confirm the EDR already quarantined it, tag the case, and close. Analysts are drowning. What should you build, and how far should automation go?

  • A) A playbook that auto-enriches and auto-closes these cases end to end with no analyst interaction
  • B) A playbook that auto-enriches and tags but routes every case to an analyst for a manual close decision
  • C) A new YARA-L rule that suppresses these alerts entirely so they never create cases
  • D) A dashboard that counts the alerts so leadership can decide whether to hire more analysts
Reveal answer & explanation

Correct: A. These steps are deterministic, low-risk, and already remediated by the EDR, so full automation of enrich-confirm-tag-close is the textbook case for hands-off SOAR, freeing analysts for judgement work.

B: Forcing a manual close on a fully deterministic, already-contained workflow keeps the analyst burden the scenario is trying to remove. C: Suppressing the alerts blinds you to the malware activity and its trend entirely. D: A dashboard measures the problem but does nothing to reduce the repetitive toil.

Q3. After a ransomware near-miss, the incident commander wants the response repeatable and the lessons captured so the next occurrence is faster and cheaper. Which action best closes the incident lifecycle?

  • A) Immediately raise the severity of all ransomware-related detections to maximum across the tenant
  • B) Archive the case and move on, since containment succeeded and no data was lost
  • C) Run a post-incident review and feed findings into new detections, reference lists, and playbook updates
  • D) Network-isolate the affected hosts again to make sure the threat has not returned
Reveal answer & explanation

Correct: C. The lessons-learned phase turns a single response into durable improvement by feeding detections, reference lists, and playbooks, which is exactly what makes the next incident faster and cheaper.

A: Blanket-maxing severity creates alert fatigue and isn't a structured improvement. B: Archiving without review wastes the most valuable output of an incident — the lessons. D: Re-isolating already-contained hosts is redundant containment, not lifecycle closure or improvement.
