Section 1~25%

Designing & Planning the Architecture

The largest section. You translate business and technical requirements into a Google Cloud design, choosing compute, storage, networking, and AI building blocks — always anchored in the Well-Architected Framework.

Requirements: functional vs non-functional

Every architecture question starts with requirements. Functional requirements describe what the system must do (e.g., "process orders"); non-functional requirements describe how well — latency, availability, durability, RTO/RPO, compliance, and cost. The exam loves to bury a single decisive constraint in the stem (a latency SLA, a data-residency rule, a budget cap). Find it first; it usually eliminates two options.

  • Tie the design to measurable success: KPIs, ROI, SLOs.
  • Always produce a business continuity plan — what happens when a region fails?
  • Workload disposition for migrations: rehost, replatform, refactor, rebuild, retire (a.k.a. build / buy / modify / deprecate).
⚠️ Exam Trap: When two answers both technically work, the correct one is the one that best satisfies the explicit business constraint in the scenario — not the most powerful or most modern service.

The Well-Architected Framework lens

The framework's six pillars are the examiner's value system. Score every option against them:

  • Operational excellence — automation, observability, measurable SLOs.
  • Security — least privilege, defence in depth.
  • Reliability — HA, failover, graceful degradation.
  • Performance optimization — right-sized, low-latency.
  • Cost optimization — pay for what you need.
  • Sustainability — efficient resource use.
Pillars "Some Old People Really Crave Sustainability" — Security, Operational excellence, Performance, Reliability, Cost, Sustainability.

Choosing compute and storage

Compute: Cloud Run for stateless containers and fastest delivery; GKE for complex microservices needing fine control or service mesh; Compute Engine for lift-and-shift, licensing, GPUs, or VMware Engine; Cloud Run functions for event glue. Spot VMs only for fault-tolerant batch.

Storage: match the access pattern — Cloud Storage (objects/data lake), Filestore (POSIX), Cloud SQL/AlloyDB (relational OLTP), Spanner (global, strongly-consistent relational), Bigtable (high-throughput wide-column NoSQL), Firestore (document/mobile), BigQuery (analytics).

⚠️ Exam Trap: "Strongly consistent AND global" is almost always Spanner. Cloud SQL cross-region replicas are asynchronous (RPO > 0).

Migration planning

Use Migration Center for automated discovery, dependency mapping, fit assessment, and TCO before any move. Then pick the mover: Migrate to Virtual Machines for VMs, Database Migration Service for databases, Storage Transfer Service / Transfer Appliance for data at rest. Plan waves, test workloads, and account for software-license implications.

Designing for HA & DR (RTO/RPO)

Two numbers drive every resilience design: RTO (how fast you must recover) and RPO (how much data loss you can tolerate). They pick the DR pattern:

  • Backup & restore — cheapest, highest RTO/RPO (hours).
  • Cold / warm standby (pilot light) — minimal idle resources, faster recovery.
  • Hot standby / multi-region active-active — near-zero RTO/RPO, highest cost.

Spread across zones for HA within a region; span regions for disaster recovery. Use regional managed instance groups, multi-region storage, and a load balancer with health checks for automatic failover.

⚠️ Exam Trap: "Near-zero RPO across regions" rules out async replication (Cloud SQL cross-region replicas) and backup/restore — you need synchronous/multi-region (Spanner) or active-active.

🧠 Warm-Up Quiz

Q1. A retailer needs globally consistent inventory counts with zero RPO across three continents. Which managed database?

Reveal answer

Cloud Spanner (multi-region).

Only Spanner gives synchronous, externally consistent global writes with RPO 0; Cloud SQL/AlloyDB cross-region replicas are asynchronous.

Q2. A two-person team wants to ship a stateless container with zero cost when idle and no infrastructure to manage. What compute?

Reveal answer

Cloud Run.

Scale-to-zero, request autoscaling, fully managed — minimal ops, no idle cost.

Q3. Two designs both meet the technical need, but one costs less and one is more "cutting edge". The stem names a hard budget cap. Which wins?

Reveal answer

The one that satisfies the explicit budget constraint.

The keyed answer matches the stated business constraint, not the most powerful or newest service.

Q4. A fault-tolerant nightly batch job wants the lowest compute cost. What VM option?

Reveal answer

Spot VMs.

Deeply discounted and fine for interruptible, restartable batch work; never for latency-critical stateful services.

📝 Exam-Style Practice

Q1. A global retailer must keep inventory counts strongly consistent across the US, Europe, and Asia with an RPO of zero, and the application is relational. Which database should the architecture use?

  • A) Cloud SQL with cross-region read replicas promoted on failover.
  • B) Cloud Spanner configured as a multi-region instance.
  • C) Bigtable replicated across three regions.
  • D) Firestore in multi-region mode with offline sync.
Reveal answer & explanation

Correct: B. Spanner is the only relational option giving synchronous, globally consistent writes with RPO 0 at multi-region scale.

A: Cross-region replicas are asynchronous, so RPO > 0. C: Bigtable is wide-column NoSQL, not relational, and replication is eventually consistent. D: Firestore is a document store and not the fit for relational inventory with RPO 0.

Q2. An architecture review lists two valid designs. One uses a brand-new managed service; the other is simpler and clearly cheaper. The scenario states a strict monthly budget. Which design should you recommend?

  • A) The newer service, because modern services are favoured.
  • B) The cheaper design that still meets every functional requirement and the budget.
  • C) Whichever has the highest raw performance.
  • D) A hybrid that runs both to hedge risk.
Reveal answer & explanation

Correct: B. The keyed answer satisfies the explicit business constraint — here, the budget — while meeting the functional needs.

A/C: "Newer" or "fastest" isn't the goal when a budget is stated. D: Running both raises cost and complexity, breaking the constraint.

Q3. You are planning a migration of 300 mixed VMs and databases. Leadership wants discovery, dependency mapping, fit assessment, and a TCO estimate before committing. What should you start with?

  • A) Migrate to Virtual Machines for an immediate lift-and-shift of everything.
  • B) Migration Center for discovery, dependency mapping, fit assessment, and TCO.
  • C) Database Migration Service to move the databases first.
  • D) Transfer Appliance to ship the data, then decide.
Reveal answer & explanation

Correct: B. Migration Center is the assessment-and-planning tool: it discovers assets, maps dependencies, assesses fit, and estimates TCO before any move.

A/C: These are execution movers, used after assessment, not for planning. D: Transfer Appliance moves bulk data; it doesn't assess or plan the migration.
