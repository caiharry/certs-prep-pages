Section 4~15%

Preparing & Using Data for Analysis

Make data fast for BI, ready for ML, and safely shareable.

BI acceleration & access

Speed BI with BI Engine and materialized views; pre-calculate fields; troubleshoot slow queries. Connect tools like Connected Sheets (spreadsheet access to BigQuery) and Looker Studio. Protect data with column-level security, data masking, IAM, and DLP.

⚠️ Exam Trap: Business users who want to analyse BigQuery data in a spreadsheet need Connected Sheets — Looker Studio is a dashboard, not a sheet.

Data for AI/ML and sharing

Prepare features for training/serving (incl. BigQuery ML), and prepare unstructured data for embeddings and RAG. Share governed data with Analytics Hub (publish/subscribe to datasets) rather than copying.

⚠️ Exam Trap: Share BigQuery data across teams with least operational overhead via Analytics Hub, not by replicating tables into each project.

🧠 Warm-Up Quiz

Q1. Non-technical users want to wrangle data with a GUI then analyse it in a spreadsheet. Which combo?

Reveal answer

Dataprep + Connected Sheets.

GUI wrangling; spreadsheet access to BigQuery results.

Q2. Let many teams query each other's BigQuery data with minimal ops and no duplication. Which feature?

Reveal answer

Analytics Hub (BigQuery sharing).

Publish/subscribe to shared datasets without copies.
