Section 1~32%

Designing Scalable, Secure, Reliable Apps

The heavyweight third of the exam: choose the right platform, design secure applications, and pick the correct data store and schema.

Platform selection

Match the runtime to the workload:

  • Cloud Run — stateless containers, scale-to-zero, per-instance concurrency, fastest path.
  • GKE — complex or stateful microservices, sidecars, service mesh, fine control.
  • Compute Engine — lift-and-shift, licensing, GPUs.

Add caching with Memorystore, decouple with Pub/Sub, and route events with Eventarc. Orchestrate multi-step flows with Workflows, schedule with Cloud Scheduler, and queue per-task work with Cloud Tasks.

⚠️ Exam Trap: When the scenario stresses minimal operational overhead or fastest delivery, the answer is almost always the most managed option — usually Cloud Run.

The authentication ladder

Authenticate to Google APIs the recommended way, in order of preference:

  • Application Default Credentials (ADC) with an attached service account — no keys to manage.
  • Workload Identity Federation for external/CI workloads — keyless.
  • Long-lived service-account keys only as a last resort (rotate, store in Secret Manager).

Store secrets in Secret Manager, protect images with Binary Authorization, and run every service with least privilege.

⚠️ Exam Trap: Distributing service-account keys to clients or embedding API keys for IAM-protected services is a guaranteed wrong answer.

Storing & accessing data

Pick by consistency and shape: Spanner (global strong consistency), AlloyDB (Postgres-compatible HTAP), Cloud SQL (regional managed MySQL/Postgres), Bigtable (high-throughput wide-column), Firestore (document/mobile sync). Grant time-limited object access with signed URLs rather than making buckets public.

⚠️ Exam Trap: To let a client upload/download one object for a limited time with no IAM change, use a signed URL.

🧠 Warm-Up Quiz

Q1. A stateless API container must scale to zero and need no node management. Which platform?

Reveal answer

Cloud Run.

Managed, scale-to-zero, request autoscaling.

Q2. Your Cloud Run service must call Cloud Storage with the least credential burden. What do you use?

Reveal answer

An attached service account via Application Default Credentials.

No keys to manage or rotate; library picks up ADC automatically.
