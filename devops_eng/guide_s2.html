Section 2~25%

Building & Implementing CI/CD Pipelines

Joint-largest section: design and run CI/CD for apps, infra, and ML, with a secured software supply chain.

CI/CD building blocks

Cloud Build for CI (build, test, containerise); Cloud Deploy for CD to GKE/Cloud Run using Skaffold and Kustomize; Artifact Registry to store images and packages. Add approval gates and pipeline triggers; integrate third-party tools (Git, Jenkins, Argo CD) where needed.

Deployment strategies

Choose by risk tolerance: canary (small % first, metric-gated ramp), blue/green (instant cutover with fast rollback), rolling, and traffic splitting / feature flags. Define success metrics from app/ML telemetry to drive promotion.

⚠️ Exam Trap: Need gradual exposure with metric-gated promotion and quick rollback? That's canary, not blue/green (which flips 100% at once).

Securing the pipeline

Run Artifact Analysis vulnerability scanning, enforce Binary Authorization and the SLSA framework for supply-chain integrity, and scope IAM by environment. Inject secrets at build vs runtime via Secret Manager, Certificate Manager, and KMS.

Supply chain Binary Authorization + SLSA = only verified, attested artifacts reach production.

🧠 Warm-Up Quiz

Q1. Release to 5% of users, watch metrics, then ramp with rollback. Which strategy?

Reveal answer

Canary deployment.

Gradual, metric-gated, fast rollback.

Q2. Continuous delivery to GKE/Cloud Run with Skaffold and approvals. Which service?

Reveal answer

Cloud Deploy.

Managed CD with release promotion and approval gates.
