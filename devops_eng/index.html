📚 Exam Sections at a Glance

Section 1 — ~20%

Bootstrapping & Maintaining an Organization

Resource hierarchy, shared networking, IaC, and secure cloud dev environments.

  • Resource hierarchy, folders, projects & org policies
  • Shared VPC, VPC Peering, Private Service Connect
  • IaC: Infrastructure Manager, Config Connector, Terraform, GitOps
  • Cloud Workstations, Cloud Shell & Gemini Code Assist
Open Study Guide →
Section 2 — ~25%

Building CI/CD Pipelines

Continuous integration, delivery, and a secured software supply chain.

  • CI with Cloud Build; CD with Cloud Deploy (Skaffold, Kustomize)
  • Artifact Registry & artifact analysis
  • Deployment strategies: canary, blue/green, rolling, traffic split
  • Binary Authorization & SLSA supply-chain security
Open Study Guide →
Section 3 — ~18%

Site Reliability Engineering

Balance velocity and reliability with SLIs, SLOs, and error budgets.

  • SLIs vs SLOs vs SLAs and the number of "nines"
  • Error budgets & error-budget policy
  • Capacity planning, quotas & autoscaling (MIGs, Cloud Run, GKE)
  • Incident mitigation: drain, add capacity, rollback
Open Study Guide →
Section 4 — ~25%

Observability & Troubleshooting

Instrument telemetry, analyse logs/metrics/traces, and find root cause.

  • Ops Agent, OpenTelemetry, Managed Service for Prometheus
  • Logs Explorer query language, log-based & custom metrics
  • Dashboards, alerting policies & synthetic monitors
  • Distributed traces, spans & trace/log correlation

Section 5 — ~12%

Optimizing Performance & Cost

Apply FinOps practices and recommenders to trim spend without hurting reliability.

  • Active Assist insights & Google Cloud recommenders
  • Spot VMs, CUDs & sustained-use discounts
  • Network tiers & observability cost control
  • Per-workload optimisation: GKE, Cloud Run, Compute Engine

💡 Coach's Study Tip!

Sections 2 and 4 are 50% of the exam combined — master Cloud Build/Cloud Deploy pipelines and the Cloud Observability stack (Logging, Monitoring, Trace) first. SRE concepts (SLI/SLO/error budget) are small in weight but show up inside CI/CD and observability questions, so learn them early. Leave FinOps recommenders for a final-day skim.
