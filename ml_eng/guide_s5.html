Section 5~22%

Automating & Orchestrating Pipelines

The largest section: build reproducible end-to-end pipelines, automate retraining, and track metadata.

Orchestration choice

Vertex AI Pipelines (KFP) is the default — serverless, ML-native, reproducible, with built-in lineage via Vertex ML Metadata. Use Cloud Composer (Airflow) only when you need broad cross-system workflow orchestration. TFX components suit TensorFlow pipelines.

⚠️ Exam Trap: For an ML-specific, serverless, reproducible pipeline pick Vertex AI Pipelines; Composer is for general workflow orchestration.

Retraining & lineage

Trigger retraining on a schedule or on drift/skew signals, and deploy via CI/CD (Cloud Build). Track artifact versions and data/model lineage in Vertex ML Metadata so every model is reproducible and auditable.

🧠 Warm-Up Quiz

Q1. Need a serverless, reproducible ML pipeline with artifact lineage. Which orchestrator?

Reveal answer

Vertex AI Pipelines.

ML-native, serverless, integrates with Vertex ML Metadata.

Q2. What records model artifact versions and data lineage on Vertex AI?

Reveal answer

Vertex ML Metadata.

Tracks lineage and versions for reproducibility.
