Theme 3: Hybrid & Multicloud Connectivity

Interconnect, VPN, Cloud Router, BGP, and hybrid DNS

Theme 3 🟢

3.1 Cloud Interconnect: The Heavy Artillery

When you need massive, reliable, enterprise-grade bandwidth between your data center and Google Cloud, standard VPNs won't cut it. You need Cloud Interconnect. Let's break down the flavors and SLAs: 🔌

🚀 Dedicated vs. Partner vs. Cross-Cloud Interconnect

  • Dedicated Interconnect: You rent a physical 10G or 100G fiber connection directly from your hardware to a Google co-location facility. Supports MACsec (Layer 2 encryption) for ultra-secure hardware-level encryption!
  • Partner Interconnect: Best when your data center is not in a Google co-location facility. You connect to a service provider (like Equinix or Megaport) who already has a pipe to Google. Supports connections from 50 Mbps up to 50 Gbps.
    • Layer 2 Partner: You configure the BGP session on your own on-prem routers.
    • Layer 3 Partner: The service provider manages the BGP session and routes traffic for you.
  • Cross-Cloud Interconnect: Connects Google Cloud directly to other cloud providers (AWS, Azure, OCI) over high-speed physical links without traffic routing over the public internet! ☁️
The 99.99% SLA Interconnect Topology Trap! To get Google's 99.99% SLA, you MUST have a highly available topology consisting of:
  1. Two physical Dedicated Interconnect connections in Metro A (Zone 1 and Zone 2).
  2. Two physical Dedicated Interconnect connections in Metro B (Zone 1 and Zone 2).
  3. Four VLAN attachments (two in each metro) routed to two separate Cloud Routers in your VPC.
If you only use a single metro (even with redundant connections in Zone 1 and Zone 2), you only get a 99.9% SLA! Examiners love to test if you can spot this difference.

3.2 Site-to-Site IPSec VPN

If you don't have the budget or time for Interconnect, Cloud VPN is your go-to. It tunnels encrypted traffic over the public internet using IPSec.

🔒 HA VPN vs. Classic VPN

Google Cloud strongly recommends HA VPN (High Availability VPN) and has deprecated many Classic VPN features. Always choose HA VPN on the exam!

  • HA VPN: Provides a 99.99% SLA by default. It uses a single gateway with two external interfaces (Interface 0 and Interface 1), each with its own public IP. To achieve the SLA, you must configure two tunnels (one from each interface) to your on-premises gateway. It strictly uses dynamic routing (BGP).
  • Classic VPN: Only supports a 99.9% SLA, uses a single interface, and supports static routing (policy-based or route-based). It is legacy and should be avoided!
  • HA VPN over Interconnect: What if you want the massive bandwidth of Interconnect and the end-to-end encryption of IPSec? You deploy HA VPN gateways inside your VPC and route their tunnels over your physical Interconnect VLAN attachments. Best of both worlds! 🛡️

3.3 Cloud Router & BGP: The Traffic Directors

Cloud Router manages dynamic routing for your VPC using BGP (Border Gateway Protocol). To control whether traffic enters or leaves GCP via a specific path, you must manipulate BGP attributes:

🚦 Controlling Inbound & Outbound Traffic Flow

  • Controlling Outbound Traffic (GCP to On-Prem):
    • MED (Multi-Exit Discriminator): When your on-premises routers advertise routes to Cloud Router, they attach a MED value (metric). Cloud Router prefers the path with the lowest MED.
    • Route Priority / Cost: You can configure Cloud Router to assign a lower priority (which translates to a higher BGP metric) to certain learned routes.
  • Controlling Inbound Traffic (On-Prem to GCP):
    • AS-Path Prepending: You can configure Cloud Router to prepend its own Autonomous System Number (ASN) multiple times when advertising routes to on-premises. On-premises routers will see a longer AS-Path and prefer the other, shorter path!
    • Custom Advertised Routes: Instead of advertising all subnets, you can configure Cloud Router to advertise only specific CIDR blocks to on-premises to restrict traffic flow.
Best Path Selection: Legacy vs. Standard Mode!

This is a high-yield dynamic routing configuration on your VPC network:

  • Legacy Mode (Default): Processes AS-path length comparisons only within a single BGP task. This means if you have multiple Cloud Routers or BGP sessions on different tasks, Cloud Router might not consistently select the shortest AS-path across them!
  • Standard Mode: Adheres strictly to RFC 4271. It evaluates and compares AS-path lengths consistently across all learned routes network-wide. It also supports advanced features like bgpAlwaysCompareMed (always compare Multi-Exit Discriminators across different neighbor ASNs) and custom handling of inter-region costs.
Exam Tip: For large enterprise deployments with complex multi-region hybrid routing, recommend switching the VPC routing configuration to Standard Mode to ensure predictable, RFC-compliant BGP path selection!
"Why did the Cloud Router get promoted? Because it always made standard, RFC-compliant path selections and never showed any legacy bias!" 📈

3.4 NCC Hybrid Spokes & Router Appliances

With Network Connectivity Center (NCC), you can attach HA VPN tunnels and Interconnect VLAN attachments as Hybrid Spokes. This unlocks site-to-site data transfer!

If you have a branch office in London and a data center in New York, both connected to GCP via HA VPN, they can route traffic to each other through Google's global fiber network by enabling site-to-site data transfer on their NCC spokes! This turns Google's network into your private WAN. 🌐

Additionally, you can deploy third-party SD-WAN or firewall VMs as Router Appliances (RAs). These VMs peer with Cloud Router over BGP to dynamically exchange routes, allowing seamless integration of third-party routing intelligence into your GCP VPC.

🧠 Quick-Fire Warm-Up Quiz!

Test your recall before diving into the proper practice exam. No peeking!

Q1: What is the minimum number of physical Dedicated Interconnect connections required to achieve a 99.99% SLA?

Four connections! You need two connections in Metro A (Zone 1 and Zone 2) and two connections in Metro B (Zone 1 and Zone 2) to survive a full metro outage.

Q2: Which BGP attribute should you use to make your on-premises routers prefer HA VPN Tunnel A over HA VPN Tunnel B for traffic entering Google Cloud?

AS-Path Prepending! By configuring Cloud Router to prepend its ASN on Tunnel B's advertisements, on-premises routers will see Tunnel B as having a longer path and will prefer Tunnel A.

Q3: What is the difference between Legacy and Standard Best Path Selection in Cloud Router?

Legacy mode only compares AS-path lengths within a single BGP task, leading to potential routing inconsistencies across different routers. Standard mode complies with RFC 4271, consistently comparing AS-path lengths network-wide.

Q4: Which encryption technology can you use to secure a Dedicated Interconnect link at Layer 2 (hardware level)?

MACsec (Media Access Control Security). It encrypts traffic directly over the physical Dedicated Interconnect link between your router and Google's edge router.

📝 Theme 3 Practice Exam Questions

Let's see if you can handle these actual exam-style hybrid connectivity brain busters!

Question 1: You are designing a high-availability hybrid network for a retail giant. The business demands a 99.99% availability SLA for their connectivity to Google Cloud. They have co-location space in a single carrier hotel in Chicago. What is the most cost-effective and compliant design?

  • A) Deploy two Dedicated Interconnect connections in Chicago (Metro A) mapped to separate edge cages, and configure four VLAN attachments distributed across different availability domains in the same carrier hotel.
  • B) Deploy two Dedicated Interconnect connections in Chicago (Metro A) to handle high-throughput transactional flows, and configure an active-active HA VPN connection over the public internet to act as a low-latency fallback path.
  • C) Deploy two Dedicated Interconnect connections in Chicago (Metro A) and two Dedicated Interconnect connections in Ashburn (Metro B), with VLAN attachments from each metro routed to separate Cloud Routers in your VPC.
  • D) Deploy a single Dedicated Interconnect connection in Chicago with 100 Gbps of bandwidth, and establish a Classic VPN tunnel mesh to handle secondary egress routing during physical maintenance windows.

Correct Answer: C

Why Option C is correct: To achieve a 99.99% SLA for Cloud Interconnect, Google strictly requires a multi-metro topology (two connections in Metro A and two connections in Metro B). A single metro topology (Option A or B) only qualifies for a 99.9% SLA, even if you have backup VPNs! 🔌

  • A: Redundant connections in a single metro only qualify for a 99.9% SLA because they cannot survive a full metro-level disaster, regardless of cage or VLAN distribution.
  • B: While using HA VPN as a backup is a great hybrid design, it does not elevate the Interconnect topology itself to a 99.99% SLA.
  • D: A single Interconnect connection has zero SLA, and Classic VPN is deprecated, representing a highly fragile design with no SLA guarantees.

Question 2: You have established an HA VPN connection between your on-premises data center and your Google Cloud VPC. You have two tunnels active. You notice that egress traffic from GCP to on-premises is asymmetric, with packets randomly choosing either tunnel, resulting in out-of-order packet delivery and firewall drops on-premises. You want to force GCP to send all traffic over Tunnel A, using Tunnel B strictly as a standby. How should you configure this?

  • A) Configure dynamic static routes in your VPC routing tables with a lower metric (priority 100) pointing to Tunnel A, and a higher metric (priority 1000) pointing to Tunnel B to manage dynamic failover paths.
  • B) On your on-premises VPN gateway, advertise your local subnets to Cloud Router over BGP, attaching a lower MED (metric) on the BGP session for Tunnel A, and a higher MED on the BGP session for Tunnel B.
  • C) On your on-premises VPN gateway, configure BGP path prepending to repeat your local Autonomous System Number (ASN) multiple times when advertising routes to Google Cloud over the Tunnel A BGP session.
  • D) Configure the dynamic routing mode of the central VPC network to Regional, which automatically restricts learned routes to a single tunnel interface unless BGP graceful restart is triggered on Tunnel B.

Correct Answer: B

Why Option B is correct: To control outbound traffic from GCP to on-premises, you must influence Cloud Router's path selection. Cloud Router evaluates the MED (metric) of learned BGP routes. By advertising a lower MED on Tunnel A, Cloud Router will prefer Tunnel A for all outbound routing, using Tunnel B only if Tunnel A's BGP session goes down. 🚦

  • A: HA VPN strictly requires dynamic routing (BGP); static routes should not be used to manage failover, and "dynamic static routes" do not exist in GCP.
  • C: AS-Path prepending on Tunnel A would make GCP avoid Tunnel A, forcing traffic onto Tunnel B instead by making the path appear longer.
  • D: Changing the dynamic routing mode to Regional does not solve asymmetric routing between two tunnels in the same region, as both tunnels are still in the same regional scope.

Question 3: You are designing a global network for a multinational corporation with VPC subnets in us-east1 and europe-west1. You have established a Dedicated Interconnect connection in Virginia (us-east1). You want VMs in europe-west1 to be able to access your on-premises network dynamically over this Interconnect. What VPC configuration is required?

  • A) Establish an internal VPC Network Peering link between the regional subnets in us-east1 and europe-west1, and configure the peering settings to allow custom transit route sharing.
  • B) Configure the VPC's dynamic routing mode to Global.
  • C) Deploy a second redundant Partner Interconnect connection in Europe (europe-west1) and configure the local Cloud Router to establish a BGP loopback interface to Virginia.
  • D) Configure custom static routes in the European subnet pointing to the Virginia Cloud Router's IP address with the next-hop set to the private service connect (PSC) gateway.

Correct Answer: B

Why Option B is correct: By setting the VPC's dynamic routing mode to Global, Cloud Routers in any region (like us-east1) will propagate their learned on-premises routes to all subnets network-wide, including those in europe-west1. This allows cross-region hybrid routing natively. 🌐

  • A: Subnets in the same VPC do not need peering; they are already part of the same global network, and peering regional subnets in the same VPC is impossible.
  • C: While a second Interconnect in Europe is great for redundancy, it is not strictly required just to enable cross-region routing and dynamic BGP loopbacks across regions are not supported over Interconnect.
  • D: Static routes are complex to maintain, do not support dynamic BGP failover, and PSC cannot be used as a next-hop gateway for raw VM-to-on-prem subnet routing.

Question 4: You are designing a complex multi-region network with redundant Cloud Routers in us-east1 and us-west1. You notice that during a simulated failover, Cloud Router occasionally routes traffic to a longer AS-path route instead of a shorter one because the BGP sessions are handled by different internal router tasks. How can you ensure consistent, RFC-compliant BGP path selection across your entire VPC?

  • A) Switch the entire hybrid infrastructure to static routes with configured route metrics on both Cloud Routers to bypass the BGP dynamic route convergence process.
  • B) Configure the VPC network's BGP best path selection mode to Standard.
  • C) Enable Bidirectional Forwarding Detection (BFD) with a 100ms interval on all active BGP sessions to synchronize the internal router tasks during failover.
  • D) Set the VPC dynamic routing mode of the central network to Regional, which prevents Cloud Router from propagating BGP dynamic routes across regional boundaries during a task restart.

Correct Answer: B

Why Option B is correct: By default, GCP VPCs use Legacy best path selection, which evaluates AS-paths only within a single BGP task, leading to potential inconsistencies across separate routers. Changing the VPC's BGP best path selection mode to Standard forces Cloud Router to strictly comply with RFC 4271, ensuring consistent, predictable AS-path length comparison across all BGP sessions network-wide! 📈

  • A: Static routing does not scale, lacks dynamic failover capabilities, and is not a valid approach for large-scale enterprise hybrid networks.
  • C: BFD detects link failures faster (sub-second), but it does not alter the BGP best path selection algorithm or task-level route evaluation.
  • D: Regional dynamic routing restricts route propagation to a single region, which does not solve multi-region path selection inconsistencies and breaks global connectivity.
