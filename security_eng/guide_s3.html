Section 3~23%

Ensuring Data Protection

Protect data at rest, in transit, and in use; secure secrets and AI workloads.

Encryption key models

  • Google-managed — default.
  • CMEK (Cloud KMS) — you control rotation/revocation; supports import.
  • Cloud EKM — keys remain in an external/on-prem HSM, never in Google Cloud.
  • Confidential Computing — encrypts data in use (memory).
⚠️ Exam Trap: "Keys must never leave our HSM" = Cloud EKM. "Generate then import" = Cloud KMS import.

Sensitive data & secrets

Sensitive Data Protection (Cloud DLP) discovers, classifies, redacts, and pseudonymises PII. Secret Manager stores versioned secrets with IAM control and rotation. Apply Cloud Storage object lifecycle policies and restrict access to data services (BigQuery, Cloud SQL, GCS).

Securing AI workloads

Apply security and privacy controls to ML systems: protect training data and models from exfiltration, secure Vertex AI access, and consider Model Armor / Sensitive Data Protection for generative AI inputs and outputs.

🧠 Warm-Up Quiz

Q1. Discover and redact credit-card numbers in a Cloud Storage dataset before analysis. Which service?

Reveal answer

Sensitive Data Protection (Cloud DLP).

Detects, classifies, and de-identifies sensitive data.

Q2. Regulator requires keys held only in your on-prem HSM. Which option?

Reveal answer

Cloud EKM.

External keys never persist in Google Cloud.
