Section 4~19%

Managing Operations

Automate security, detect threats, and respond — the SecOps backbone of this exam.

Automation & supply chain

Scan for CVEs in CI/CD, enforce Binary Authorization so only signed images deploy to GKE/Cloud Run, automate image hardening and patch management, and manage drift with custom organization policies and Security Health Analytics custom modules.

Logging, monitoring & detection

Analyse network logs (VPC Flow Logs, Cloud NGFW, Packet Mirroring, Cloud IDS, Log Analytics). Configure Cloud Audit Logs and data-access logs, route them with log sinks / aggregated sinks (to BigQuery/Pub/Sub/GCS or external SIEM), and centralise findings in Security Command Center.

⚠️ Exam Trap: Data-access logs are not all enabled by default — you must explicitly turn them on for the services you need to audit.

Audit logs: the four types & routing them

Know the four Cloud Audit Logs streams and which are on by default. Admin Activity and System Event logs are always on and free. Data Access logs (who read/wrote data) and Policy Denied logs are not all enabled by default — you turn Data Access logging on per service in the IAM audit config. To retain or analyse logs beyond the default _Default bucket, create log sinks; an aggregated sink at the org/folder level captures logs from every child project in one place and exports to BigQuery, Pub/Sub, Cloud Storage, or an external SIEM.

  • Admin Activity — config changes; always on, cannot disable.
  • Data Access — reads/writes of data; mostly off by default, enable per service.
  • Aggregated sink — org/folder-level export of all child logs.
⚠️ Exam Trap: "We had no record of who read the data" almost always means Data Access logs were never enabled — they are off by default. Don't blame the sink; fix the IAM audit config first.

Threat detection & incident response

Security Command Center is the posture and threat hub: Security Health Analytics flags misconfigurations, Event Threat Detection spots malicious activity in logs, and Container/VM Threat Detection watch workloads. For network forensics, VPC Flow Logs show connection metadata, Packet Mirroring feeds full packets to Cloud IDS for signature-based detection. When responding, you can quarantine a VM with a deny-all firewall tag, rotate or disable compromised credentials/keys, and snapshot disks for evidence — automate the routine steps so response is fast and repeatable.

  • Security Health Analytics — misconfiguration findings.
  • Event Threat Detection — log-based threat detection.
  • Cloud IDS — packet-mirrored, signature-based intrusion detection.
SCC = SEE Security Command Center is where you see it all — misconfig (SHA), threats (ETD), and posture in one console.

Reveal answer

Binary Authorization.

Q2. Centralise org-wide security findings and posture in one place. Which service?

Reveal answer

Security Command Center.

Q3. You need every project's logs exported to one central SIEM with the least ongoing maintenance. What do you configure?

Reveal answer

An aggregated log sink at the organization (or folder) level.

It captures logs from all current and future child projects without per-project setup.

Q4. An investigation finds no record of which user read a sensitive BigQuery table last month. What was most likely missing?

Reveal answer

Data Access audit logs were not enabled for BigQuery.

Data Access logs are off by default and must be turned on per service in the IAM audit config.

📝 Exam-Style Practice

Q1. A compliance team needs a tamper-resistant record of every read against objects in a set of regulated Cloud Storage buckets, retained for seven years in an external SIEM. Currently nothing is recorded. What is the correct first step before any export will help?

  • A) Create an aggregated log sink to the SIEM and assume the read events will start flowing
  • B) Enable Data Access audit logs for Cloud Storage in the IAM audit configuration
  • C) Turn on VPC Flow Logs on the subnets that host the storage clients
  • D) Add a Security Health Analytics custom module that watches the buckets for reads
Reveal answer & explanation

Correct: B. Object reads are Data Access events, which are off by default; until you enable Data Access audit logs for Cloud Storage there is nothing for any sink to export.

A: A sink only routes logs that already exist — with Data Access logging off there is nothing to route. C: Flow Logs record network connection metadata, not who read which storage object. D: Security Health Analytics reports misconfigurations, not per-object read events.

Q2. Event Threat Detection raises a finding that a Compute Engine instance is communicating with a known crypto-mining command-and-control IP. You must contain the host immediately while preserving evidence for later analysis. What is the best response?

  • A) Delete the instance right away so the mining process can no longer run anywhere
  • B) Remove the service account from the instance and wait to see if the traffic stops on its own
  • C) Apply a deny-all firewall rule via a network tag to isolate the VM, then snapshot its disks
  • D) Lower the instance's machine type to starve the miner of CPU while you investigate
Reveal answer & explanation

Correct: C. Isolating the VM with a deny-all firewall tag cuts the C2 channel immediately, and snapshotting the disks preserves forensic evidence before any further change.

A: Deleting the instance destroys exactly the evidence you are told to preserve. B: Removing the service account does not stop outbound network traffic and "wait and see" is not containment. D: Resizing slows the miner but leaves the C2 connection open, so the host is not contained.

Q3. Security wants signature-based intrusion detection on east-west traffic between application tiers in a VPC, with full-payload inspection and managed threat signatures, without writing detection rules themselves. Which combination fits?

  • A) Packet Mirroring feeding Cloud IDS, which applies Google-managed threat signatures
  • B) VPC Flow Logs exported to BigQuery with scheduled queries flagging suspicious patterns
  • C) Cloud Armor rules inspecting the traffic between the application tiers
  • D) A custom Security Health Analytics module scanning the tier configurations for drift
Reveal answer & explanation

Correct: A. Cloud IDS consumes mirrored packets and applies Google-managed (Palo Alto) threat signatures for full-payload, signature-based detection with no rule authoring required.

B: Flow Logs carry only connection metadata, not payloads, and require you to write the detection queries yourself. C: Cloud Armor protects inbound HTTP(S) at the load balancer edge, not east-west VPC traffic. D: Security Health Analytics checks configuration posture, not live payload-level intrusions.
