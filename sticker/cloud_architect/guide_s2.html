Section 2~17.5%

Managing & Provisioning Infrastructure

Stand up network topologies, storage systems, and compute — including the newer Gemini Enterprise Agent Platform for ML workflows.

Hybrid & multicloud connectivity

Pick the on-ramp by SLA, bandwidth, and encryption needs:

  • HA VPN — 99.99% SLA, IPsec-encrypted over the internet, fast to deploy.
  • Dedicated Interconnect — 10/100 Gbps, private, lowest latency, needs a colo cross-connect; not encrypted by default.
  • Partner Interconnect — via a service provider when you have no colo presence.
  • Cloud Router provides dynamic BGP for all of the above.
⚠️ Exam Trap: Interconnect is not encrypted in transit by default — layer HA VPN over it or use MACsec when encryption is mandated.

Shared VPC vs VPC Peering vs PSC

Shared VPC centralises network admin in a host project while service projects attach — best for org-wide governance. VPC Peering connects two VPCs as equals but is non-transitive. Private Service Connect exposes a single service privately across the boundary.

⚠️ Exam Trap: VPC Peering is non-transitive: A–B and B–C does not give A–C connectivity.

Compute provisioning & AI platform

Configure volatility (spot vs standard), patch management, and orchestration (MIGs, GKE). The current exam adds the Gemini Enterprise Agent Platform: Agent Platform Pipelines orchestrate the ML lifecycle, and AI Hypercomputer runs large-scale GPU/TPU training. Know that Model Garden supplies models and prebuilt AI APIs (Vision, Document AI, etc.) cover standard tasks.

Infrastructure as Code & provisioning

Provision repeatably, never by clicking in the console. Terraform is the de facto IaC tool on Google Cloud; the Cloud Foundation Toolkit ships opinionated, secure module blueprints, and Config Controller / Config Connector manage GCP resources declaratively through the Kubernetes API for GitOps. Keep state remote and locked, review changes via pull requests, and bake guardrails (org policies) into the pipeline.

⚠️ Exam Trap: "Repeatable, reviewable, auditable environment creation" = Terraform/IaC in a pipeline — not gcloud scripts run by hand or console clicks.
IaC If it isn't in version control, it didn't happen — declarative + peer-reviewed beats imperative + ad-hoc.

🧠 Warm-Up Quiz

Q1. You need an encrypted, 99.99%-SLA connection to on-prem live this week, with no colocation presence. What do you choose?

Reveal answer

HA VPN (two tunnels).

Encrypted, 99.99% SLA, internet-based, fast; Dedicated Interconnect needs a colo and isn't encrypted by default.

Q2. Central network team must own one network used by many project teams. Which model?

Reveal answer

Shared VPC.

Host project holds the network; service projects attach — centralised governance.

Q3. VPC A peers with B, and B peers with C. Can A reach C?

Reveal answer

No — VPC Peering is non-transitive.

Each peering connects only the two directly-peered VPCs; A–C needs its own peering, an NCC hub, or a transit design.

Q4. Auditors require every environment to be created the same way, reviewed before apply. What approach?

Reveal answer

Infrastructure as Code (Terraform) through a CI/CD pipeline.

Declarative, version-controlled, peer-reviewed, and auditable — unlike manual console changes.

📝 Exam-Style Practice

Q1. A regulated enterprise needs a connection to on-premises that is encrypted in transit, carries a 99.99% SLA, and can be live this week. They have no colocation presence yet. What should they deploy?

  • A) Dedicated Interconnect with two connections in one metro.
  • B) HA VPN with two tunnels and Cloud Router BGP.
  • C) Partner Interconnect with a single VLAN attachment.
  • D) A Classic VPN tunnel to a single on-prem gateway.
Reveal answer & explanation

Correct: B. HA VPN is encrypted, carries a 99.99% SLA, and deploys in days over the internet — matching every constraint.

A/C: Interconnect needs colo provisioning (weeks) and isn't encrypted by default. D: Classic VPN is deprecated and has no 99.99% SLA.

Q2. A central platform team must own and govern one network that dozens of application teams' projects deploy into, while keeping firewall and routing control centralised. Which design fits best?

  • A) A separate VPC per team, fully meshed with VPC Peering.
  • B) Shared VPC, with the network in a host project and teams' service projects attached.
  • C) One giant project that every team shares directly.
  • D) Private Service Connect endpoints between every pair of teams.
Reveal answer & explanation

Correct: B. Shared VPC centralises network and firewall control in the host project while service projects attach — exactly the governance model described.

A: A peering mesh is non-transitive and hard to govern centrally. C: One shared project destroys project-level isolation and billing. D: PSC exposes single services, not a shared network.

Q3. After several outages caused by hand-made console changes, leadership wants all infrastructure changes to be declarative, peer-reviewed, and auditable. What should the team adopt?

  • A) A shared runbook of gcloud commands operators paste into the console.
  • B) Terraform managed through a CI/CD pipeline with remote state and pull-request review.
  • C) Nightly snapshots so changes can be rolled back after the fact.
  • D) Tighter IAM so only senior engineers can click in the console.
Reveal answer & explanation

Correct: B. IaC in a pipeline makes changes declarative, version-controlled, reviewed, and auditable — eliminating ad-hoc drift.

A: Pasting commands is still manual and error-prone. C: Snapshots recover data, not change governance. D: Restricting who clicks doesn't make changes reviewable or repeatable.
